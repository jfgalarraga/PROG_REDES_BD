Ethernet: Estándares y funcionamiento

Ethernet (IEEE 802.3)

El protocolo más utilizado en redes LAN cableadas, que ofrece alta velocidad y confiabilidad mediante cables de cobre o fibra óptica.

Wi-Fi (IEEE 802.11)

Protocolo estándar para redes LAN inalámbricas que permite la transmisión de datos sin cables, usando ondas de radio.

DNS (Domain Name System)

Protocolo utilizado para la resolución de nombres de dominio a direcciones IP, facilitando la conectividad en redes LAN e Internet.

Protocolo TCP/IP

TCP/IP es un conjunto de protocolos que facilitan la comunicación entre dispositivos en una red. TCP se ocupa de garantizar que los datos lleguen correctamente, mientras que IP se encarga de dirigir esos datos hacia el destino adecuado a través de una dirección única.

🕸️ Capa de Acceso a la Red

La capa de acceso a la red se encarga de la transmisión física de los datos a través del medio de comunicación, ya sea un cable (como en Ethernet) o señales inalámbricas (como en Wi-Fi).

Protocolos:

ARP

Protocolo utilizado para mapear direcciones de red (IP) a direcciones físicas (MAC) dentro de una red local.

Wi-Fi

Utilizado en redes LAN inalámbricas. Define cómo los dispositivos se comunican sin cables, proporcionando una forma de acceder a la red mediante señales de radio.

Ethernet

Define el formato de los paquetes de datos y las reglas de acceso al medio para evitar colisiones.

📶 Capa de Internet

La capa de Internet se encarga de dirigir los paquetes de datos de un dispositivo a otro a través de la red, utilizando el protocolo IP (Internet Protocol). Esta capa es responsable del direccionamiento y el enrutamiento de los paquetes de datos, garantizando que lleguen al destino correcto.

Direcciones IP:

IPv4

IPv6

Tipos de IP:

IP Privada

Es una dirección IP que se utiliza dentro de una red privada, como una red local (LAN), y no es accesible directamente desde Internet.

IP Pública

Es una dirección única asignada a un dispositivo o red que está directamente accesible a través de Internet.

💻 Capa de Transporte

La capa de transporte asegura que los datos se entreguen de manera fiable entre los dispositivos de una red, garantizando que la comunicación de extremo a extremo sea adecuada.

Protocolos:

TCP

Es un protocolo orientado a la conexión y confiable. Establece una conexión entre el emisor y el receptor antes de la transmisión de datos.

UDP

UDP envía los datos como datagramas, sin preocuparse por la ordenación ni confirmación. Esto permite una transmisión más rápida, pero también más susceptible a pérdidas o errores en la entrega.

🖧 Capa de Aplicación

En esta capa residen los protocolos que interactúan directamente con las aplicaciones del usuario y proporcionan los servicios necesarios para la comunicación de datos entre los dispositivos.

Protocolos:

HTTP

Es el protocolo utilizado para la transferencia de páginas web. Cuando accedes a una página en tu navegador, este protocolo se utiliza para enviar solicitudes y recibir respuestas entre tu navegador y el servidor web.

FTP

Protocolo utilizado para la transferencia de archivos entre un cliente y un servidor a través de una red. FTP permite a los usuarios subir o descargar archivos desde servidores remotos.

DNS

Este protocolo traduce los nombres de dominio legibles por humanos (como www.ejemplo.com) en direcciones IP, que son utilizadas para enrutar paquetes de datos en la red.

SMTP

Protocolo utilizado para el envío de correos electrónicos entre servidores. Cuando envías un correo desde un cliente de correo, SMTP es el encargado de enviar ese mensaje al servidor de destino.
